# Доступ к приложению из интернета (туннель)

Чтобы открыть приложение по публичному URL (не только по локальной сети), используйте туннель. Достаточно **одного** туннеля на порт **5173**: фронт проксирует запросы `/api` на бэкенд.

## Вариант 1: ngrok (рекомендуется)

1. **Установите ngrok**  
   https://ngrok.com/download  
   Или: `winget install ngrok.ngrok` / скачать exe и положить в PATH.

2. **Запустите приложение** (если ещё не запущено):
   ```bash
   docker compose up -d
   ```

3. **Поднимите туннель** (в отдельном терминале на этом же ПК):
   ```bash
   ngrok http 5173
   ```

4. В выводе ngrok будет строка вида:
   ```
   Forwarding   https://xxxx-xx-xx-xx-xx.ngrok-free.app -> http://localhost:5173
   ```
   Откройте **этот https-адрес** в браузере — с телефона или любого устройства с интернетом.

5. (Опционально) Зарегистрируйтесь на ngrok.com и в консоли выполните `ngrok config add-authtoken <token>` — тогда лимиты выше и URL стабильнее.

---

## Вариант 2: Cloudflare Tunnel (cloudflared)

1. Скачайте [cloudflared](https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/install-and-setup/installation/).
2. Запуск туннеля:
   ```bash
   cloudflared tunnel --url http://localhost:5173
   ```
3. В выводе будет временный URL вида `https://xxx.trycloudflare.com` — откройте его в браузере.

---

## Вариант 3: Проброс портов на роутере

Если нужен доступ по вашему внешнему IP без сторонних сервисов:

1. В веб-интерфейсе роутера найдите «Проброс портов» / «Port Forwarding».
2. Создайте правило: внешний порт **5173** → внутренний IP **192.168.40.189**, порт **5173** (TCP).
3. Узнайте свой внешний IP: в браузере откройте https://whatismyip.com (или аналог).
4. С другого устройства в интернете откройте `http://<ваш_внешний_IP>:5173`.

**Минус:** открываете порт в интернет; для демо лучше туннель (ngrok/cloudflared).
